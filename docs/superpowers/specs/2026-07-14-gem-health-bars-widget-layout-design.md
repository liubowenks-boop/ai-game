# 宝石血条与 HUD 自动对齐设计

**日期：** 2026-07-14
**目标环境：** Cocos Creator 3.8.8，720 × 1280 竖屏设计画布

## 目标

- 使用用户提供的两张新图片替换城门耐久框和 Boss 血条框。
- 把运行时血量填充改为矩形宝石质感，不重新生成 UI 图片。
- 上移 Boss 血条，缩小并对齐“剩余敌人”，把暂停、自动、统计组成连续的右上按钮组。
- 使用 Cocos `Widget` 和 `Layout` 表达屏幕贴边与同组排列关系，减少互相独立的魔法坐标。
- 保留当前未提交的 HUD 50% 缩放成果，不改变英雄头像、“三选一强化卡”和战斗循环。

## 已确认的视觉方向

血条采用代码绘制的分层矩形方案：

- Boss 血条始终使用红宝石色；
- 城门耐久根据数值使用祖母绿、黄玉、红宝石三档颜色；
- 每个血条由深色底槽、主色层、顶部亮面、底部暗面、窄高光和少量矩形切面组成；
- 所有绘制区域均使用直角矩形，不使用当前的圆角药丸形状；
- 数值标签继续显示在血条内部，并始终处于宝石填充层之上。

选择该方案是因为它能准确响应实时数值，视觉层次比纯色矩形丰富，同时不引入额外遮罩纹理、Shader 或运行时资源依赖。

### Widget 与 Layout 结构

右侧控制区使用以下节点层级：

```text
TopHudLayer
└── RightControlStack  (Widget: Top + Right)
    ├── PauseResumeHud
    ├── AutoHud
    └── StatisticsHud
```

- `RightControlStack` 使用 `Widget` 的 Top、Right 对齐，边距从配置读取；
- Widget 的对齐模式使用窗口尺寸变化时更新的模式，避免无必要的逐帧对齐；
- 容器添加垂直 `Layout`，方向为 `TOP_TO_BOTTOM`，子节点顺序决定按钮顺序；
- Layout 使用容器自适应模式，容器高度由三个按钮和间距计算；
- 单个按钮不再设置互不相关的绝对 `y` 坐标；
- 点击区域保持独立，三个按钮在视觉上连接但不互相覆盖。

这遵循 Cocos 官方文档中“贴边小元素使用 Widget”和“同组子节点由 Layout 自动排列”的建议，同时适配项目实际使用的 Cocos Creator 3.8.8 API。

## 宝石血条绘制

### 公共几何

Boss 与城门使用同一个矩形绘制器，输入为轨道配置、比例和调色板。比例先被限制在 `0...1`；无效数值按 0 处理。

绘制顺序：

1. 内槽阴影：接近黑褐色的完整矩形；
2. 宝石暗底：按当前比例绘制深色矩形；
3. 主色层：在暗底内部缩进 1–2px；
4. 顶部亮面：占填充高度约 30%，使用较亮、略透明的颜色；
5. 底部暗面：占填充高度约 25%，增强切割感；
6. 高光线：在上方绘制 1–2px 的浅色矩形；
7. 切面块：按固定宽度绘制少量交替明暗矩形，最后一块宽度不得超过当前填充宽度。

比例为 0 时只显示内槽；极低血量时至少显示 1px 主色，避免非零数值完全不可见。所有子矩形都限制在当前填充宽度内，不越过框体。

### 配色

调色板集中放入 `BattleHudConfig.ts`：

- Boss：固定红宝石，主色为深红到绯红，亮面偏橙红，高光偏暖白；
- 城门 `ratio > 0.55`：祖母绿；
- 城门 `0.28 < ratio <= 0.55`：黄玉；
- 城门 `ratio <= 0.28`：红宝石。

阈值沿用当前逻辑，色值改为包含 `base/main/highlight/shadow/glint/facet` 的完整调色板。以后调整色彩不需要修改绘制函数。

### 新框体内轨道

新素材的奖章、标签和金属边框保留，动态血条只覆盖右侧深色槽。首轮轨道配置为：

- Boss：中心偏移约 `(20, -2)`，宽约 `315`，高约 `22`；
- 城门：中心偏移约 `(54, -1)`，宽约 `378`，高约 `24`。

轨道具体数值在处理后的透明裁图导入后，通过 Cocos 预览校准，并继续集中保存在 `BattleHudConfig.ts`。

## 代码结构

- `tools/prepare-battle-hud-assets.py`
  - 接受本次两张 Downloads 图片作为指定输入；
  - 复用去白底、裁剪、缩放、UUID 保留与清单更新逻辑。
- `assets/scripts/ui/BattleHudConfig.ts`
  - 保存所有 HUD 矩形、右侧按钮组 Widget/Layout 参数、轨道几何、字体大小、阈值和宝石调色板；
  - 继续作为用户后续调整位置和尺寸的唯一主要入口。
- `assets/scripts/ui/BattleHudView.ts`
  - 创建右侧按钮容器及其 Widget/Layout；
  - 创建三个按钮为容器子节点；
  - 使用通用矩形宝石绘制器刷新 Boss 和城门进度。
- `tools/battle-hud-logic.test.ts`
  - 验证布局边界、对齐关系、按钮组配置和既有 HUD 文案逻辑。
- `tools/battle-hud-assets.test.py`
  - 验证处理后的素材透明边角、尺寸、清单和 `.meta` 完整性。

现有英雄头像和三选一强化卡的创建、布局与资源路径不改动。

## 容错与兼容

- 新框图加载失败时，数值文字和点击区域仍存在，不恢复已经替换的旧框图；
- Boss 不存在时，标题、框体、填充与百分比一起隐藏；
- `Widget` 和 `Layout` 仅负责右侧按钮组，不约束动态血条宽度；
- 处理脚本重复执行必须得到相同输出，不重复创建 UUID；
- 当前工作区未提交的 HUD 50% 缩放修改将一起纳入最终实现，不覆盖或丢弃。

## 用户调整入口

完成后，主要配置文件仍为：

`assets/scripts/ui/BattleHudConfig.ts`

- 修改 `layout.*.x/y`：调整普通 HUD 元素位置；
- 修改 `layout.*.width/height`：调整元素尺寸；
- 修改 `rightControls.right/top`：整体移动暂停、自动、统计按钮组；
- 修改 `rightControls.itemWidth/itemHeight/spacing`：调整按钮单元和连接间距；
- 修改 `tracks.boss` 或 `tracks.city`：调整框体内部血条的位置与尺寸；
- 修改 `gemPalettes` 和 `cityThresholds`：调整宝石颜色与城门变色阈值。
