# 参考城墙、五英雄布局与四系受击特效设计

## 目标

在 Cocos Creator 3.8.8、720x1280 竖屏战斗中完成四项视觉升级：

- 将靠近城墙的左右废墟分别向屏幕外侧移动设计宽度的 8%。
- 使用用户提供的图 1 原始城墙像素替换当前城墙，保持门、旗帜、砖块、宽高比、镜头角度和纵向体量。
- 将场上五名英雄统一放大 30%，重新校准脚线、遮挡与间距。
- 按图 2 的视觉语言重做火焰、雷电、毒/咒术和金色物理四套受击粒子特效。

同时将六格英雄头像栏收敛为五格，并将城墙血条移动到五头像栏正上方。

## 已确认方向

- 城墙路线：直接精确提取图 1，不重新生成城墙。
- 人物比例：五名英雄统一放大 30%。
- 血条位置：移动到英雄头像栏正上方。
- 头像数量：六格改为五格。
- 受击特效路线：方案 B，使用 `gpt-image-2` 生成四套独立高分辨率资源。
- 元素映射：火焰爆燃、蓝紫雷爆、绿色毒符爆裂、金色星爆。
- 游戏逻辑：不改变伤害、攻速、敌人移动、攻击目标或关卡节奏。

## 城墙资源契约

### 输入与提取

图 1 是唯一城墙像素来源。处理过程必须满足：

- 只删除与画布边界连通的白色背景，不按全局亮度删除白色像素。
- 保留砖石高光、门饰金属高光、旗帜金纹和抗锯齿边缘。
- 不使用生成式模型补画、重绘、扩边或改变城墙细节。
- 对主体包围盒做统一比例缩放，禁止分别拉伸 X/Y。
- 缩放后主体宽度为 720px，纵向高度由原始宽高比唯一决定。

### 前后层拆分

输出继续使用现有文件：

| 文件 | 画布 | 责任 |
| --- | ---: | --- |
| `battle_wall_back.png` | 720x480 RGBA | 后沿垛口、完整顶面和位于人物后方的墙体内容 |
| `battle_wall_front.png` | 720x340 RGBA | 人物前方的前沿垛口、前墙、城门、旗帜和链条遮挡 |

两张图使用同一缩放比例、同一 X 原点和同一世界坐标基准。叠加后必须无错位、接缝、重复砖块或透明裂缝。现有 `.meta` 文件和 UUID 保持不变，避免场景与运行时引用漂移。

### 加载与回退

- `wallBack` 与 `wallFront` 都是必需层；任一层缺失都不能显示不完整城墙。
- 新城墙任一必需资源加载失败时，回退到 `battle_bg_sandgate_720x1280.png` 的旧版完整战场，不阻断战斗。
- 失败最多记录一次警告，不创建纯色节点或程序绘制替代物。

## 地形布局

设计分辨率保持 720x1280，原点保持屏幕中心。

### 废墟

当前左右废墟中心为 `x=-180/180`。8% 按设计宽度计算：

```text
720 * 0.08 = 57.6px
```

新位置为：

| 图层 | 当前 X | 新 X |
| --- | ---: | ---: |
| `TerrainRuinsLeft` | -180 | -237.6 |
| `TerrainRuinsRight` | 180 | 237.6 |

Y、尺寸、透明度和渲染顺序不变。部分废墟允许自然超出屏幕边缘，以扩大城墙前方与中央道路的净空。

### 城墙与人物脚线

- 城墙主体等比铺满 720px，不做纵向压缩。
- `wallBackY`、`wallFrontY` 和 `unitY` 以新资源叠加后的可行走顶面为同一基准校准。
- 五名英雄共享同一脚线，脚底落在顶面内部，不得悬空或陷入砖面。
- `CityWallFront` 只遮挡鞋底到小腿下部；头部、武器、施法动作和起手特效必须完整可见。
- 地面与城墙的分隔由新墙体透明边缘完成，不使用额外实色遮罩。

## 五英雄阵型

场上仍只有五名英雄：固定雷法师、三名普通英雄和居中主角。

| 顺序 | X | 角色 |
| --- | ---: | --- |
| 1 | -240 | 固定雷法师 |
| 2 | -120 | 普通英雄槽 0 |
| 3 | 0 | 固定主角 |
| 4 | 120 | 普通英雄槽 1 |
| 5 | 240 | 普通英雄槽 2 |

视觉尺寸规则：

- 主角、雷法师和普通英雄的最终显示尺寸统一乘以 `1.3`。
- 缩放围绕各自脚部锚点进行，不能让脚线随缩放上下漂移。
- 主角保持中心和最高视觉识别度，但不再通过额外尺寸差破坏五人头顶对齐。
- 相邻人物的透明边界可轻微接近，但非透明主体不得重叠。
- 攻击、受击和待机动画只能在角色内部做姿态偏移，不改变阵型中心。

## 四系受击特效

### 资源

新增资源放入 `assets/bundles/ui/battle_fx_common/`：

| 文件 | 尺寸 | 用途 |
| --- | ---: | --- |
| `fx_v3_hit_fire_eruption.png` | 512x512 RGBA | 红橙地裂爆燃与黑红岩片 |
| `fx_v3_hit_thunder_crater.png` | 512x512 RGBA | 蓝紫雷柱、白热爆心和晶体碎片 |
| `fx_v3_hit_poison_talisman.png` | 512x512 RGBA | 绿色毒爆、符箓和黑绿碎片 |
| `fx_v3_hit_gold_starburst.png` | 512x512 RGBA | 金色放射光刺、冲击环和碎石 |

全部资源使用 `gpt-image-2` 单独生成：

- 火焰、雷电、金色使用纯绿色键背景。
- 绿色毒符使用纯洋红键背景，避免与主体颜色冲突。
- 背景必须平坦、无阴影、无渐变、无地面、无文字和水印。
- 本地移除色键后检查透明四角、半透明辉光、白热爆心和边缘色溢。
- 正式资源不得包含参考图中的白色画布。

### 职业映射

| 受击类型 | 来源 |
| --- | --- |
| 火焰爆燃 | 主角、火药师、灼烧伤害 |
| 蓝紫雷爆 | 雷法师、连锁闪电 |
| 绿色毒符爆裂 | 毒师、咒术师、毒/咒术状态伤害 |
| 金色星爆 | 弓手、护卫、鼓手及无明确元素的物理命中 |

治疗效果不播放敌人受击爆裂，继续使用独立治疗表现。

### 运行时分层

每次命中由以下层组成：

1. 512x512 主体纹理，负责主要识别轮廓。
2. 同色低透明冲击辉光，负责扩大瞬时亮度。
3. `ParticleSystem2D` 碎屑层，按元素发射岩片、晶体、符箓或金色碎石。
4. 0.15-0.3 秒冲击环，快速扩张并淡出。

现有弹道、起手闪光和拖尾不变，只替换 `impactTexture` 与命中粒子参数。继续使用现有对象池、关键特效优先级和并发预算。

### 特效失败处理

- 四张 V3 资源预加载失败时，按元素回退到当前已验证的 V2 命中纹理。
- 不得回退到 `Graphics`、纯色矩形、程序射线或实心圆。
- 单个可选碎屑层失败时仍播放主体纹理，不影响伤害和攻击动画。
- 每种缺失资源最多警告一次。

## 组件边界

### `BattleTerrainConfig.ts`

负责废墟位置、城墙锚点和五人脚线，不处理资源抠图或 HUD。

### `BattleTerrainPresentation.ts`

继续负责加载与排列城墙前后层，维持角色夹在两层之间的渲染顺序。

### `BattleUiLayout.ts`

负责城墙血条、放置文字和五头像矩形的唯一布局坐标。

### `BattleController.ts` 与 `GridPlacementSystem.ts`

分别负责主角/固定角色缩放和普通英雄缩放，读取统一的 1.3 视觉比例与脚线，不复制硬编码坐标。

### `BattleVfxConfig.ts`

新增四张 V3 纹理 ID、职业映射与分层参数；攻击系统只通过预设选择表现。

### `BattleVfxSystem.ts`

继续负责 UUID 预载、Sprite/ParticleSystem2D 对象池、主体/辉光/碎屑播放和 V2 回退，不计算伤害。

## 渲染顺序

从后向前：

```text
TerrainBase / TerrainRoad
TerrainRuinsLeft / TerrainRuinsRight
TerrainAtmosphereBack
CityWallBack
Enemies
WallUnitsLayer
CityWallFront
PlayerAndCompanionProjectiles
BattleFeedbackLayer
MidStatusLayer
BottomHudLayer: placement text -> city health bar -> five-avatar rail
TopHudLayer / UpgradePanelLayer
```

受击主体和粒子位于 `BattleFeedbackLayer`，不得被城墙前沿遮挡；血条位于 HUD 层，不受战斗层缩放影响。
