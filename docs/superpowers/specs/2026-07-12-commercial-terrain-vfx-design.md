# 写实重型城墙与商用战斗特效设计

## 目标

在现有 Cocos Creator 3.8.8 竖屏战斗中，参考用户提供的荒原城堡图与高清元素光效图，完成第二轮地形、城墙、角色站位和战斗特效升级。最终画面应接近写实传奇题材手游的投放素材质量，同时维持怪物、血条、弹道与伤害反馈的可读性。

本轮不更换主角或雷法师 Spine 资源，不改变五人总上限、底部六头像栏和主角攻速联动公式。

## 已确认的视觉方向

- 地形采用参考图优先的写实重型城堡方案，而非缩小城墙换取更多战场空间的平衡方案。
- 城墙约占画面下方 `32%–35%`，完整表现黑灰石砖、城门、锁链、垛口、金属包边和暗红长旗。
- 五名角色站在城墙垛口顶面，城墙前片自然遮挡人物脚部。
- 取消人物脚下现有圆圈、描边和常驻高亮框。
- 特效采用全程高亮、最大华丽度方案，并通过对象池、同屏预算和生命周期控制移动端性能。
- 主角、固定雷法师以及所有可招募英雄均配置职业化弹道和命中特效。

## 地形与城墙

### 构图

战场继续使用 `720×1280` 竖屏设计分辨率。上方和中部为纵深荒原战场，下方为重型城堡：

- 中央道路使用磨损石板、车辙和高亮尘土形成纵深引导。
- 两侧布置木栅、碎石、骨骸、破损器械和暗红军旗，但不侵占三条主要行军通道。
- 远景增加低对比营寨、烟尘和暖灰天空，避免与怪物抢夺视觉焦点。
- 城墙使用大块深灰石砖、青铜门饰、锁链、垛口和长旗，材质细节与参考图一致。

### 模块化资源

沿用现有七层 2D Sprite 架构，重新生成并替换以下内容：

1. 地形底图。
2. 中央道路覆盖层。
3. 左侧废墟覆盖层。
4. 右侧废墟覆盖层。
5. 远景气氛层。
6. 城墙后片。
7. 城墙前片。

城墙前后片必须带透明通道。人物、站位逻辑和弹道插入既有稳定渲染根节点，不把整张参考图作为不可拆分背景。

必需层加载失败时继续回退旧背景；可选装饰层失败时只隐藏对应层，不阻断战斗。

## 人物站位与防线

- 角色顺序保持：雷法师、普通英雄、主角、普通英雄、普通英雄。
- 五人继续使用 `x=-240/-120/0/120/240` 的单排中心线；最终 `y` 可随新垛口顶面做小幅统一校准，但不得破坏横向对齐。
- 取消 `GridPlacementSystem` 的常驻圆形底板、描边和主角圆形光环。
- 空站位不显示圆框。待放置时只在可用垛口显示短暂金色符文微光或边缘亮光；放置结束后立即淡出。
- 当前红色防线在任何状态下都不再绘制。
- 逻辑防线从 `cityLineY=-290` 前移到 `cityLineY=-235`。怪物在城墙和人物前方约 `85px` 处停下并攻击，不穿入人物或墙体。
- 城墙受击反馈使用石屑、尘雾、暖色震光和城池血条，不恢复红线。

## 战斗特效系统

### 资源策略

采用 AI 生成分层美术与 Cocos 2D 粒子混合路线。使用用户提供的光效图作为颜色、亮度和轮廓参考，生成项目专用透明高清纹理，不直接复制参考图中的成品元素。

预期纹理类别：

- 赤金火焰刀芒与火焰碎片。
- 蓝紫主雷、白色电芯和分支电弧。
- 金色穿刺箭头、速度线和星芒。
- 冰晶、毒雾、治疗灵气和护盾冲击。
- 通用冲击环、闪光核心、火花、烟尘和石屑。

优先全部自产；只有无法满足的缺口才引入明确标记为 CC0 的外部素材，并在仓库记录来源、许可证和原始下载地址。

### 运行时架构

新增统一的 `BattleVfxSystem`，负责：

- 预载并缓存光效 SpriteFrame。
- 根据元素与事件创建蓄力、弹道、拖尾、命中、受击和死亡效果。
- 管理 Sprite、`ParticleSystem2D` 和短时 Graphics 冲击波的组合。
- 对粒子与临时节点进行对象池复用。
- 对资源加载失败进行 Graphics 降级，不阻断战斗。
- 限制同屏粒子和特效节点数量，并在重开战斗时彻底回收。

实现以 Creator 3.8.8 API 为准。`ParticleSystem2D` 运行时设置 `custom`、`spriteFrame`、容量、生命周期、颜色、速度、加速度和发射范围，通过 `resetSystem()` 播放，通过 `stopSystem()` 和对象池回收。Creator 4.0 文档只作为视觉参数和粒子概念参考，不使用 4.0 独有序列化结构。

官方参考：

- [Creator 4.0 2D 粒子系统](https://docs.cocos.com/creator/4.0/manual/zh/particle-system/2d-particle/2d-particle.html)
- [Creator 3.8 ParticleSystem2D API](https://docs.cocos.com/creator/3.8/api/zh/class/ParticleSystem2D?id=PositionType)

### 职业表现

- 主角：赤金武器蓄光、弧形斩焰、金色高速弹道、火花和环形冲击波。
- 雷法师：蓝紫主闪电、白色电芯、多分支电弧、落点电爆和短暂地面电痕。
- 弓手：锐利金色箭轨和星芒命中。
- 火药师：赤橙爆焰、火星和烟尘。
- 冰法师：蓝白冰晶轨迹、碎冰和霜雾。
- 毒师与咒术师：绿紫符咒、毒雾和暗色残留。
- 护卫：金属冲击、盾光和石屑。
- 鼓手与治疗师：暖金增益波纹、青绿治疗灵气和柔和光点。

怪物受击增加短促白闪、方向性火花、元素残留和分层伤害数字；死亡使用向外爆散和淡出，不使用整屏闪白。

### 华丽度与性能

特效保持高亮和较长尾焰：

- 普通命中约 `40–70` 粒子。
- 暴击或 Boss 命中约 `90–140` 粒子。
- 普通效果生命周期约 `0.18–0.65s`，大型冲击效果可延长到 `0.9s`。
- 同屏粒子与节点设置硬上限；超限时优先回收最旧的非关键装饰效果。
- 高频弹道使用对象池，禁止每次攻击无限创建永久节点。

目标为竖屏 Web 预览接近 `60 FPS`，连续运行时节点数量稳定。

## 数值调整

- 雷法师基础攻击间隔由 `0.6s` 调整为 `0.85s`。
- 雷法攻击动画继续按实际攻击间隔计算播放速度。
- 普通怪物基础生命由 `20` 调整为 `24`，提高约 `20%`。
- 普通怪物基础移动速度由 `34` 调整为 `30`，降低约 `12%`。
- 其他敌人类型继续通过现有倍率派生，不在本轮单独重写全部敌人配置。

## 测试与验收

### 自动检查

- 地形与光效图片尺寸、透明通道和透明边角。
- 资源与 `.meta` 成对存在，UUID 清单与文件一致。
- 圆形站位底板和主角常驻圆环不再绘制。
- `cityLineY=-235`，红线绘制入口不存在。
- 雷法攻击间隔、敌人基础生命和移动速度符合新值。
- 粒子对象池、同屏预算、资源失败降级和重开清理具有回归测试。
- 现有 MVP、地形、UI、动画、Spine 和类型检查继续通过。

### Cocos 视觉验收

在 Cocos Creator 3.8.8 内置浏览器以 `720×1280` 运行：

- 空阵型和五人满编均无脚下圆圈。
- 城墙构图、材质、人物站位与用户参考图基本一致。
- 怪物在城墙前停止，不与人物或墙体重叠。
- 主角、雷法师和全部可招募英雄的职业弹道、命中和受击效果可见。
- 高亮连发时怪物、血条和伤害数字仍可辨认。
- 连续运行至少 30 秒，检查帧率、节点累积、闪屏和控制台 warning/error。
- 保存空阵型、五人满编和高强度战斗截图，与两张参考图做视觉对比。
