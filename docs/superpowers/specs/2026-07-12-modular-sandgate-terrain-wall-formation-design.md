# 模块化荒原地形与城墙五人阵型设计

## 目标

在不改变《英雄令》现有荒原战争主题与 Cocos Creator 3.8.8 技术栈的前提下，替换当前单张战场背景，制作可独立调整的模块化 2D 地形；将城墙整体向画面下方移动约 80px，并让场上最多 5 名角色等距站在城墙垛口上。

本设计不迁移到 Cocos Creator 4.0，也不使用真正的 `cc.Terrain`。参考文档中的 Terrain 工作流要求创建 3D 地形节点和地形资源，并通过 Sculpt、Paint、PBR 与 LOD 等能力编辑地貌，不适合直接嵌入当前 2D 竖屏战斗：

- https://docs.cocos.com/creator/4.0/manual/zh/editor/terrain/

## 已确认方向

- 技术路线：Creator 3.8.8 生成式 2D 战场。
- 资源路线：完整模块化场景，而不是单张烘焙背景。
- 美术主题：保持当前暖色荒原、战争废墟、沙土道路和石质城墙风格。
- 地形结构：宽阔中央战场，保持三路敌人直线推进，废墟退到道路两侧。
- 城墙位置：采用 W2 标准后移，约向下 80px。
- 人数口径：场上总人数最多 5，包含主角、固定雷法师和最多 3 名普通英雄。
- 阵型：主角居中、雷法师最左、三名普通英雄填充其余位置，五人等距横排。

## 范围

### 包含

- 生成并导入新的模块化荒原地形位图资源。
- 新增地形表现配置与加载组件。
- 调整城墙、防线、主角、雷法师和普通英雄槽位坐标。
- 将普通英雄槽位改为城墙单排布局。
- 将普通英雄上限固定为 3，使场上总人数不超过 5。
- 从首轮开始用召唤伤害卡替换无效的上阵数量卡。
- 保留现有战斗、攻击、动画、合成与 HUD 工作流。

### 不包含

- Creator 4.0 项目迁移。
- 3D Terrain、3D 相机、3D 物理或 3D 角色。
- 新敌人寻路或弯曲路线。
- 新角色、技能、装备或关卡系统。
- 改变荒原主题、世界观或 UI 主风格。
- 重做底部六格英雄头像栏。

## 资源契约

新资源放入 `assets/bundles/battle_common/`，保持现有 battle common 资源归属。

| 资源 | 建议尺寸 | Alpha | 责任 |
| --- | ---: | --- | --- |
| `battle_terrain_base_720x1280.png` | 720x1280 | 不透明 | 荒原基础地面、远景与整体色调 |
| `battle_road_overlay.png` | 720x1280 | 透明 | 宽阔中央三路战道与轻量地表纹理 |
| `battle_ruins_left.png` | 360x900 | 透明 | 左侧战争废墟、残旗与碎石 |
| `battle_ruins_right.png` | 360x900 | 透明 | 右侧战争废墟、拒马与碎石 |
| `battle_wall_back.png` | 720x240 | 透明 | 位于角色后方的城墙主体 |
| `battle_wall_front.png` | 720x160 | 透明 | 位于角色前方的垛口与前沿遮挡 |
| `battle_atmosphere.png` | 720x900 | 透明 | 低透明烟尘、余火和远景空气感 |

资源生成要求：

- 颜色、材质和时代感必须与当前 `battle_bg_sandgate_720x1280.png` 一致。
- 不得把角色、敌人、站位环、血条、文字或技能特效烘焙进图片。
- 道路中心保持低对比和低噪声，确保敌人、伤害数字和飞弹清晰。
- 左右废墟不得侵入三条敌人主要路径。
- 城墙主体和前沿必须分离，角色节点夹在两层之间。
- 所有透明层的边缘不得残留白底、黑底或实色矩形。
- 每个图片和目录必须包含稳定的 Cocos `.meta` 文件。

## 组件边界

### `BattleTerrainConfig.ts`

统一保存：

- 地形资源名称。
- 层级顺序。
- 城墙主体与前沿锚点。
- 防线和五人阵型坐标。
- 透明层可选/必需标记。

该文件不加载资源，也不读写战斗状态。

### `BattleTerrainPresentation.ts`

负责：

- 创建和复用地形层节点。
- 预加载 battle common 模块化资源。
- 按配置排列地形、道路、废墟、城墙和环境层。
- 暴露角色层和城墙前沿层之间的稳定插入位置。
- 处理单层缺失与完整背景回退。
- 避免重复节点、重复加载和重复警告。

该组件不计算人数、伤害、敌人移动或英雄合成。

### 现有系统调整

- `BattleController` 只负责创建 `BattleTerrainPresentation`，并将角色/敌人表现放到正确层级。
- `BattleMvpModel` 负责人数上限、槽位、邻接和防线坐标。
- `GridPlacementSystem` 负责三个普通英雄城墙槽位的交互和合成表现。
- `ThunderMagePresentation` 保留独立 Spine 与蓝白攻击效果，只读取新的固定坐标。
- `PlayerAutoAttackSystem` 保留主角金色飞弹和命中特效。

## 渲染层级

从后向前排列：

```text
TerrainBase
TerrainRoad
TerrainRuinsLeft / TerrainRuinsRight
TerrainAtmosphereBack
CityWallBack
Enemies / EnemyEffects
WallUnitBackingRings
ThunderMage / OrdinaryHeroes / MainHero
CityWallFront
PlayerAndCompanionProjectiles
BattleFeedback
MidStatusLayer / TopHudLayer / BottomHudLayer / UpgradePanelLayer
```

角色脚部位于城墙前沿后方，武器、头部和攻击动作位于前沿上方。投射物必须位于角色前方且不得被城墙前沿截断。

## 坐标与布局

设计分辨率保持 720x1280，坐标原点保持屏幕中心。

### 城墙与防线

- `cityLineY`: `-210 -> -290`
- 五人角色中心线：`y = -320`
- 城墙主体视觉锚点：约 `y = -365`
- 城墙前沿视觉锚点：按生成资源透明边界校准，覆盖角色脚部至小腿。
- 城池血条维持约 `y = -214`，位于角色上方。
- 底部头像栏维持现有位置，角色中心线与头像栏之间保留约 180px 安全间距。

### 五人位置

| 顺序 | X | Y | 所有权 |
| --- | ---: | ---: | --- |
| 1 | -240 | -320 | 固定雷法师 |
| 2 | -120 | -320 | 普通英雄槽 0 |
| 3 | 0 | -320 | 固定主角 |
| 4 | 120 | -320 | 普通英雄槽 1 |
| 5 | 240 | -320 | 普通英雄槽 2 |

站位环使用低亮度铜色圆环，空槽不显示“前1”“后2”等文字。角色动画不得改变站位环中心，只允许在节点内部做小幅姿势偏移。

## 战斗模型与人数规则

- 场上总人数上限固定为 5。
- 主角与雷法师固定占用 2 个位置，但不进入普通英雄数组。
- 普通英雄上限固定为 3。
- 普通英雄槽位为 0、1、2；固定雷法师继续使用保留槽位 3。
- 删除现有普通槽位 4，不保留不可用的幽灵槽位。
- 普通槽位邻接关系为 `0 <-> 1 <-> 2`。
- 固定雷法师槽不参与放置、替换、邻接或合成。
- 同名普通英雄仍可在同格或相邻普通槽位合成，最高等级不变。
- `summon_slots_plus_1` 从所有卡牌轮转中替换为 `summon_hero_damage_20`，不得再向玩家提供无效扩容。

## 敌人和攻击同步

- 敌人保持现有三路直线移动，不新增寻路。
- 敌人停止点、城墙受击点、近城墙目标排序统一使用新的 `cityLineY = -290`。
- 主角攻击原点移动到 `(0, -320)`。
- 雷法师攻击原点随固定节点移动到 `(-240, -320)`，其特效起点按武器位置增加局部偏移。
- 普通英雄 DPS 与站位读取三个新槽位坐标。
- 主角 Spine 攻击、雷法师 Spine 攻击、金色飞弹和蓝白雷击逻辑保持不变。

## 资源加载与失败处理

- 模块化地形只预加载一次。
- 基础地形和城墙主体是必需层；任一失败时回退到当前 `battle_bg_sandgate_720x1280.png`。
- 道路、左右废墟、城墙前沿和环境层是可选层；单层失败时跳过该层。
- 每种失败最多记录一次警告。
- 晚到的异步回调必须检查节点有效性和 presentation 所有权。
- 重开战斗不得重复创建地形节点或重复加载资源。
- 地形失败不得阻断模型 tick、敌人移动、英雄攻击或 HUD 刷新。

## 自动测试

### 模型

- 普通英雄最多 3 名。
- 主角、雷法师和普通英雄合计不超过 5 名。
- 雷法师保留槽不可放置普通英雄。
- 槽位 4 不再存在。
- 普通槽位邻接关系为 `0 <-> 1 <-> 2`。
- 上阵数量卡不再出现在卡池，召唤伤害卡保持有效。
- 防线、敌人停止点和城墙攻击均使用 `-290`。

### 布局与表现

- 五个角色中心分别为 `-240/-120/0/120/240`，共享 `y=-320`。
- 五个角色节点互不重叠。
- 角色不与城池血条、头像栏、终极按钮或右侧操作栏重叠。
- 城墙前沿层位于角色前方，投射物层位于城墙前沿前方。
- 主角和雷法师现有动画/特效源码契约继续通过。

### 资源

- 七个模块化地形资源及 `.meta` 存在。
- 基础图尺寸为 720x1280 且不透明。
- 透明层包含 Alpha 通道，四角不得为实色背景。
- 资源名、bundle 路径和配置一致。
- 必需层失败触发旧背景回退；可选层失败不触发整体回退。

## Cocos 手动验收

使用 Cocos Creator 3.8.8、720x1280 Web 预览检查：

- 主题仍是当前荒原战争风格，但地形结构明显更新。
- 中央三路战场更宽，左右废墟不遮挡敌人。
- 城墙相对当前版本向下约 80px。
- 五名角色完整站在垛口上，主角居中、雷法师最左。
- 三名普通英雄填满后不再允许第 4 名普通英雄上阵。
- 角色、城池血条和底部头像栏没有重叠。
- 角色脚部被垛口自然遮挡，头部、武器和动画不被裁切。
- 主角金色飞弹、雷法师蓝白雷击和普通英雄攻击均清晰可见。
- 新背景不存在白边、黑底、实色矩形、闪屏或异常裁切。
- 控制台没有资源、脚本、Spine 或渲染警告/错误。
