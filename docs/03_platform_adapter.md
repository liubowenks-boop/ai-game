# 03 Platform Adapter

## 设计目标

平台适配层用于隔离抖音小游戏、快手小游戏和 Web/编辑器调试环境差异。业务代码只依赖统一接口，不直接调用平台全局对象。

当前阶段只写设计文档，不实现任何平台 API 调用代码。

## IPlatformAdapter 规划

后续可规划统一接口 `IPlatformAdapter`，用于描述平台能力边界。接口可逐步覆盖：

- `login`：登录、授权、会话管理。
- `storage`：本地存储、云存储或平台存储封装。
- `share`：分享、邀请、群分享能力。
- `record`：录屏、剪辑、发布视频能力。
- `rewarded video ad`：激励视频广告加载、播放、奖励确认和失败处理。
- `analytics`：事件埋点、漏斗、关卡进度、广告转化。
- `vibration`：轻震动、重震动和平台兼容降级。

接口设计原则：

- 返回值优先使用 Promise 或统一结果对象。
- 调用方不感知 `tt`、`ks` 或浏览器差异。
- 失败结果要可区分，例如用户取消、平台不支持、网络失败、广告未加载。
- 编辑器和 Web 调试环境必须有可运行的降级实现。

## DouyinAdapter

`DouyinAdapter` 后续对应抖音小游戏环境中的 `tt` API。该适配器只在 `platform` 模块内部使用 `tt`，并负责封装登录、分享、录屏、广告、震动和埋点等能力。

## KuaishouAdapter

`KuaishouAdapter` 后续对应快手小游戏环境中的 `ks` API。该适配器只在 `platform` 模块内部使用 `ks`，并处理快手平台在登录、广告、分享、录屏等能力上的差异。

## WebAdapter

`WebAdapter` 用于浏览器和 Cocos Creator 编辑器调试。它应提供不依赖平台 SDK 的降级行为，例如：

- 登录返回本地调试用户。
- 存储使用浏览器本地存储或内存模拟。
- 分享、录屏、广告返回可控的模拟结果。
- 埋点输出到控制台或本地调试面板。
- 震动能力按浏览器支持情况降级。

## 依赖边界

不允许战斗代码、肉鸽代码、UI 业务代码或数据模块直接依赖 `tt` 或 `ks` API。所有平台能力都必须经过 `IPlatformAdapter` 规划下的统一入口访问。
